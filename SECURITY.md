# Security

`open-source-portfolio` is a static React/Vite portfolio template. It should only contain public profile data, project summaries, service links, and sanitized assets.

Do not commit private resumes, real customer data, private repository URLs, API keys, cookies, analytics tokens, or unpublished business details.

If you find a security issue or accidental private-data exposure, report it privately through the maintainer's GitHub profile. Public issues should not include secrets, private URLs, screenshots with account tokens, or internal logs.

Before publishing a fork, review:

- public links and email addresses,
- files under `public/` and `src/`,
- generated screenshots and downloadable assets,
- deployment workflow settings.
